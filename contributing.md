# Contributing to the Docs

These docs are written for GitBook and live at the root workspace level in `docs/`.

Do not place product documentation inside `Secant-Pay/`, `Secant-frontend/`, `Secant-backend/`, or `Secant-SDK/` unless the content is package-specific developer documentation.

## Structure

- `README.md` is the docs home page.
- `SUMMARY.md` controls the GitBook sidebar.
- Product, architecture, roadmap, integrations, and settlement pages should stay as separate topic files.

## Style

- Keep pages short and scannable.
- Use direct product language.
- Keep Phase 1 limited to features already built.
- Put roadmap ideas in Phase 2 or Phase 3.
- Use Mermaid for architecture diagrams when possible.
